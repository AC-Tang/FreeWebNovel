## 1. 项目概述

### 1.1 项目简介

FreeFiction（白嫖小说网）是一个前后端分离的全栈小说阅读 Web 应用。用户可以在平台上浏览、搜索、阅读小说，参与评论互动、签到、评分、加入书架等。系统还包含自动化小说采集爬虫模块和管理后台。

### 1.2 基本信息

| 项目 | 信息 |
|------|------|
| 项目名称 | FreeFiction（后端）/ UnknownSystem（前端） |
| 项目类型 | 全栈 Web 应用 |
| 开发语言 | Java 17（后端）/ JavaScript（前端） |
| 构建工具 | Maven（后端）/ Vite + npm（前端） |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis |
| IDE | IntelliJ IDEA |

### 1.3 功能全景

```
FreeFiction
├── 前台用户端
│   ├── 用户系统（注册/登录/个人中心）
│   ├── 小说浏览（首页推荐/分类/排行/书库）
│   ├── 小说阅读（阅读器/目录/设置/进度记忆）
│   ├── 搜索功能
│   ├── 社区互动（评论/段评/弹幕/评分/点赞）

│   ├── 阅读历史
```

### 2.2 技术选型理由

| 技术选型 | 理由 |
|----------|------|
| **Spring Boot 3.5** | 最新稳定版，原生支持 Jakarta EE、虚拟线程等新特性 |
| **Java 17** | LTS 长期支持，密封类/模式匹配等语言特性 |
| **MyBatis-Plus** | 相比 JPA 更灵活，自带分页/条件构造器/代码生成器 |
| **Vue 3 Composition API** | 更好的逻辑复用和 TypeScript 支持 |
| **Vite** | 比 Webpack 更快的冷启动和 HMR |
| **Pinia** | Vue 3 官方推荐的状态管理库，TypeScript 友好 |
| **JWT 双 Token** | 无状态 + 安全性 + 用户体验的平衡方案 |
| **Druid** | 阿里巴巴开源连接池，自带 SQL 监控面板 |
| **Jsoup** | Java 生态最成熟的 HTML 解析库 |
| **Redis** | 高性能缓存，支持多种数据结构 |

## 4. 数据库设计

### 4.1 数据库信息

- **数据库名**：`freefiction`
- **字符集**：`utf8mb4`
- **引擎**：InnoDB
- **数据表数量**：20+ 张

### 4.2 核心表结构

#### 4.2.1 用户表 (users)

```sql
users
├── id (BIGINT PK)           -- 用户ID
├── username (VARCHAR 50)    -- 用户名
├── password_hash (VARCHAR 255) -- BCrypt加密密码
├── email (VARCHAR 100)      -- 邮箱（唯一）
├── avatar (VARCHAR 255)     -- 头像URL
├── role_name (VARCHAR 20)   -- 角色：ROLE_USER / ROLE_ADMIN
├── status (TINYINT)         -- 状态：0-禁用 1-正常
├── created_at / updated_at  -- 时间戳
└── last_login_at (DATETIME) -- 最后登录时间
```

#### 4.2.2 书籍表 (books)

```sql
books
├── book_id (BIGINT PK)          -- 书籍ID（使用源站ID）
├── title (VARCHAR 100)          -- 书名（全文索引）
├── subtitle (VARCHAR 200)       -- 副标题
├── author (VARCHAR 50)          -- 作者
├── category_id (INT FK)         -- 分类ID → categories
├── cover (VARCHAR 255)          -- 封面图片URL
├── description (TEXT)           -- 书籍简介
├── status (TINYINT)             -- 0-下架 1-连载中 2-已完结
├── word_count (INT)             -- 总字数
├── chapter_count (INT)          -- 章节数
├── view_count (INT)             -- 点击量
├── recommend_count (INT)        -- 推荐数
├── like_count (BIGINT)          -- 收藏数
├── rating_avg (DOUBLE)          -- 平均评分(3,2)
├── rating_count (INT)           -- 评分人数
├── last_chapter_title (VARCHAR) -- 最新章节标题
├── last_chapter_time (DATETIME) -- 最新章节更新时间
├── publish_date (DATETIME)      -- 发布时间
├── recommended (TINYINT)        -- 是否首页推荐
├── carousel (VARCHAR)           -- 轮播图路径
└── created_at / updated_at      -- 时间戳

索引：
  PRIMARY KEY (book_id DESC)
  INDEX idx_title, idx_author, idx_category_id, idx_status
  INDEX idx_view_count, idx_recommend_count, idx_rating_avg
  INDEX idx_last_chapter_time, idx_publish_date
  FULLTEXT INDEX idx_title_fulltext (title)
```

#### 4.2.3 章节表 (chapters)

```sql
chapters
├── id (BIGINT PK AUTO_INCREMENT)
├── book_id (BIGINT FK)      -- 所属书籍 → books
├── title (VARCHAR 200)      -- 章节标题
├── content (LONGTEXT)       -- 章节内容
├── sort_order (INT)         -- 章节序号
├── word_count (INT)         -- 字数
├── view_count (INT)         -- 阅读次数
└── created_at / updated_at
```

#### 4.2.4 评论表 (comments)

```sql
comments
├── id (BIGINT PK)
├── user_id (BIGINT FK)      -- 评论用户 → users
├── target_type (TINYINT)    -- 评论类型：1-小说 2-章节 3-段评
├── target_id (BIGINT)       -- 目标ID（book_id 或 chapter_id）
├── content (TEXT)           -- 评论内容
├── parent_id (BIGINT)       -- 父评论ID（用于回复）
├── like_count (INT)         -- 点赞数
├── reply_count (INT)        -- 回复数
├── paragraph_index (INT)    -- 段评段落索引
├── is_spoiler (TINYINT)     -- 是否剧透
└── created_at
```

### 5.2 统一响应设计

**Result<T>**：Controller 层统一响应格式
```java
public class Result<T> {
    int code;        // 业务状态码（200/401/403/404/500等）
    String message;  // 响应消息
    T data;          // 响应数据
    AuthResponse token;  // 认证响应（登录/刷新时携带Token）
}
```

**ServiceResult<T>**：Service 层统一返回格式
```java
public class ServiceResult<T> {
    boolean success;  // 业务是否成功
    String message;   // 错误/成功消息
    T data;          // 业务数据
}
```

**ResultCode**：状态码枚举
```java
SUCCESS(200, "操作成功"),
FAILED(500, "操作失败"),
VALIDATE_FAILED(404, "参数检验失败"),
UNAUTHORIZED(401, "暂未登录或token已经过期"),
FORBIDDEN(403, "没有相关权限"),
ERROR(500, "系统错误"),
NOT_FOUND(404, "资源未找到");
```

### 5.3 全局异常处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    // NoHandlerFoundException → 404
    // NoResourceFoundException → 404
    // RuntimeException → 500
    // Exception → 500（兜底）
}
```

### 5.4 核心 Service 详解

#### AuthService（认证服务）
- **login()**：查询用户（支持邮箱/用户名） → BCrypt 密码校验 → 返回用户信息
- **register()**：校验邮箱唯一性 → BCrypt 加密密码 → 创建用户 + 默认书架

#### NovelFetchServiceImpl（小说采集服务）
- **fetchNovel(novelId, isRefresh)**：爬取信息 → IDM 下载 TXT → GBK→UTF-8 编码转换 → 章节解析 → 入库 → 更新统计
- **fetchNovels(startId, endId)**：批量采集，含进度跟踪、随机延迟反爬
- **updateNovelChapters(novelId)**：增量更新，对比本地与远程章节列表，只下载新增章节
- **refreshNovel(novelId)**：强制全量刷新
- **refreshChapter(novelId)**：重新分章，含 IDM 重新下载 HTML 信息页

#### ChapterParseServiceImpl（章节解析服务）
- 正则匹配章节标题（"第X章"、"Chapter X" 等模式）
- 按章节标题位置切分完整 TXT 文件
- 章节去重与排序

#### DeepSeekService（AI 服务）
- 普通聊天 / 带系统提示词 / 多轮对话历史
- SSE 流式响应支持（打字机效果）
- Token 估算工具

#### RedisService（Redis 缓存服务）
- 封装的缓存读写操作
- 分级缓存时间策略（CacheTimeStrategy）

### 6.4 Axios 请求拦截器设计

这是前端最核心的工具模块：

```

  2. 从 Cookie 读取 Token → 注入 Authorization Header

  2. 检查 res.code
```

## 7. 认证与安全

### 7.1 JWT 双 Token 机制

```
┌─────────────────────────────────────────────────────────┐
│                    JWT 认证流程                           │
│                                                          │
│  登录请求                                                 │
│  POST /api/user/login                                    │
│  { username/email, password }                             │
│       │                                                   │
│       ▼                                                   │
│  AuthService.login()                                      │
│  ├── 查询用户                                             │
│  ├── BCrypt.matches(password, hash)                       │
│  └── 返回用户信息                                          │
│       │                                                   │
│       ▼                                                   │
│  UsersController.login()                                  │
│  ├── JJwtUtil.generateToken(user)      ← Access Token    │
│  │   ├── 有效期: 1天 (86400000ms)                        │
│  │   ├── 载荷: userid, role                              │
│  │   └── 算法: HS256                                     │
│  └── JJwtUtil.generateRefreshToken(user)  ← Refresh Token│
│      ├── 有效期: 10天 (864000000ms)                      │
│      └── 载荷: userid, role, tokenType:"refresh"          │
│       │                                                   │
│       ▼                                                   │
│  返回: { code, message, data: user, token: {              │
│           accessToken, refreshToken } }                    │
│                                                          │
│  ───────────────────────────────────────                 │
│                                                          │
│  后续请求                                                 │
│  GET /api/xxx                                             │
│  Header: Authorization: Bearer <accessToken>              │
│       │                                                   │
│       ▼                                                   │
│  JwtAuthenticationTokenFilter                             │
│  ├── 检查 Token 黑名单                                     │
│  ├── JJwtUtil.getUserFromToken(token) → userId           │
│  ├── 加载 UserDetails                                     │
│  └── 设置 SecurityContext                                 │
│                                                          │
│  ───────────────────────────────────────                 │
│                                                          │
│  Token 过期 (401)                                         │
│       │                                                   │
│       ▼                                                   │
│  前端 Axios 响应拦截器                                     │
│  ├── 检查 refreshToken 是否存在                            │
│  ├── isRefreshing = true (防止并发刷新)                    │
│  ├── POST /api/refresh_token                              │
│  │   Header: Authorization: Bearer <refreshToken>         │
│  ├── 刷新成功                                              │
│  │   ├── 更新 Cookie 中的 Token                            │
│  │   ├── 重放队列中等待的所有请求                           │
│  │   └── isRefreshing = false                             │
│  └── 刷新失败                                              │
│      ├── 清除 Cookie + localStorage                        │
│      └── 跳转 /login                                      │
└─────────────────────────────────────────────────────────┘
```

### 7.2 安全措施汇总

| 措施 | 实现方式 |
|------|----------|
| 密码加密 | BCryptPasswordEncoder，相同明文产生不同密文（salt） |
| 无状态认证 | SessionCreationPolicy.STATELESS，服务端不存会话 |
| Token 黑名单 | Redis 维护已注销 Token，Filter 层拦截 |
| CORS 配置 | 允许跨域来源、方法、头部，携带凭证 |
| CSRF 保护 | 无状态 API 不需要，已禁用 |
| XSS 防护 | Vue 模板默认转义，Element Plus 组件安全 |
| 限流保护 | RequestLimitFilter |

## 12. 已知问题与改进方向

### 12.1 当前不足

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 单元测试覆盖不足 | 重构风险较高 | 高 |
| DeepSeekService 被注释掉 | AI 功能不可用 | 中 |
| Elasticsearch 依赖引入但未实际使用 | 搜索仍用 MySQL LIKE/FULLTEXT | 中 |
| 前端未使用 TypeScript | 类型安全性不足 | 中 |
| 爬虫与主应用耦合 | 爬虫阻塞可能影响 API 响应 | 低 |
| 配置文件含敏感信息（API Key/密码） | 安全风险 | 高 |
| 日志文件包含在项目中 | 版本控制混乱 | 低 |

### 12.2 建议改进方向

1. **安全**：将敏感配置移至环境变量或配置中心
2. **搜索**：启用 Elasticsearch 做全文搜索，提升搜索体验
3. **测试**：补充 Service 层单元测试和 Controller 层集成测试
4. **爬虫解耦**：将爬虫模块独立为微服务，通过消息队列异步处理
5. **TypeScript**：前端迁移至 TypeScript，提升代码质量
6. **CI/CD**：配置 GitHub Actions 自动化构建与测试
7. **容器化**：编写 Dockerfile + docker-compose，一键部署
